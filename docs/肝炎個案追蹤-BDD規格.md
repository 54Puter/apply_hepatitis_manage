# 肝炎個案追蹤 - BDD 規格文件

## 概述

本文件描述「肝炎個案追蹤」功能之行為驅動開發（BDD）規格，此功能將建置於既有醫療系統平台上。

---

## 功能一：病人資料檢索

### 情境：以身分證字號檢索病人

```gherkin
Feature: 病人資料檢索
  作為醫療人員
  我希望能夠透過身分證字號或病歷號檢索病人
  以便取得肝炎追蹤所需之病人基本資料

  Scenario: 以身分證字號成功檢索病人
    Given 系統已連線至病人資料庫
    And 病人資料庫中存在身分證字號 "A123456789" 的病人
    When 醫療人員於檢索欄位輸入身分證字號 "A123456789"
    And 點擊「查詢」按鈕
    Then 系統應從病人資料庫取得該病人基本資料
    And 應顯示以下欄位：
      | 欄位名稱 | 說明 |
      | 姓名 | 病人姓名 |
      | 性別 | 病人性別 |
      | 身分證號碼 | 病人身分證字號 |
      | 病歷號 | 病人病歷號 |
      | 生日 | 出生日期 |
      | 年齡 | 依生日計算之年齡 |
      | 地址 | 通訊地址 |
      | 連絡電話-手機 | 手機號碼 |
      | 連絡電話-住家 | 住家電話 |
      | 連絡電話-公司 | 公司電話 |
      | 初診日期 | 首次就診日期 |

  Scenario: 以病歷號成功檢索病人
    Given 系統已連線至病人資料庫
    And 病人資料庫中存在病歷號 "MR20240001" 的病人
    When 醫療人員於檢索欄位輸入病歷號 "MR20240001"
    And 點擊「查詢」按鈕
    Then 系統應從病人資料庫取得該病人基本資料
    And 應顯示完整基本資料欄位（同身分證字號檢索）

  Scenario: 檢索無資料之病人
    Given 系統已連線至病人資料庫
    And 病人資料庫中不存在身分證字號 "Z999999999" 的病人
    When 醫療人員於檢索欄位輸入身分證字號 "Z999999999"
    And 點擊「查詢」按鈕
    Then 系統應顯示「查無此病人資料」之提示訊息
```

## 功能二：病人基本資料與 FIB-4 計算

```gherkin
Feature: 病人基本資料與 FIB-4
  作為醫療人員
  我希望能夠檢視病人基本資料並計算 FIB-4 肝纖維化風險
  以便進行臨床評估

  Scenario: 區塊佈局為左75%右25%
    Given 醫療人員檢視病人資料區塊
    Then 左側 75% 應顯示病人基本資料
    And 右側 25% 應顯示 FIB-4 計算區塊

  Scenario: FIB-4 計算輸入與結果
    Given 已成功檢索並顯示病人基本資料
    When 醫療人員檢視 FIB-4 區塊
    Then 應顯示四個輸入欄位：年齡、AST (S-GOT)、ALT (S-GPT)、PLT (10⁹/L)
    And 系統應依病人資料自動填入數值
    And 醫療人員可手動修改輸入值
    And 修改後應自動重新計算並更新結果
    And 結果應顯示至小數點後 2 位
    And 若無法計算則顯示 "-"

  Scenario: FIB-4 風險描述
    Given FIB-4 計算結果為有效數值
    When 醫療人員檢視 FIB-4 區塊
    Then 結果下方應顯示風險描述：
      | 數值範圍 | 描述 |
      | < 1.3 | 肝纖維化低度風險 |
      | 1.3 ~ 2.67 | 肝纖維化中度風險 |
      | > 2.67 | 肝纖維化高度風險 |

  Scenario: FIB-4 計算公式
    Given 年齡、AST、ALT、PLT 皆為有效數值
    When 系統計算 FIB-4
    Then 公式應為：[年齡 × AST] / [√ALT × PLT]
    And PLT 單位為 10⁹/L
```

## 功能三：肝炎追蹤項目區塊

### 情境：肝炎追蹤項目表格顯示

```gherkin
Feature: 肝炎追蹤項目
  作為醫療人員
  我希望能夠檢視病人的肝炎相關檢驗項目最新數值
  以便進行個案追蹤與評估

  Background:
    Given 已成功檢索並顯示病人基本資料
    And 病人資料庫包含該病人之檢驗紀錄

  Scenario: 篩選按鈕與收攬狀態
    Given 醫療人員檢視肝炎追蹤區塊
    Then 應顯示三個篩選按鈕：「一般肝炎」、「HBV 追蹤」、「HCV 追蹤」
    And 點擊「一般肝炎」時：
      | 群組 | 收攬狀態 |
      | HBV 相關 | 收攬 |
      | HCV 相關 | 收攬 |
      | 肝炎追蹤 | 展開 |
      | 其他檢驗項目 | 收攬 |
    And 點擊「HBV 追蹤」時：HBV 相關、肝炎追蹤 展開；其餘收攬
    And 點擊「HCV 追蹤」時：HCV 相關、肝炎追蹤 展開；其餘收攬

  Scenario: 項目分類與可折疊列
    Given 醫療人員檢視肝炎追蹤區塊
    Then 應顯示以下可折疊分類列：
      | 分類 | 包含項目 |
      | HBV 相關 | HBsAg, HBeAg, HBV DNA, Anti-HBc, Anti-HBs, Anti-HBe |
      | HCV 相關 | Anti-HCV, HCV RNA, HCV Genotype |
      | 肝炎追蹤 | AST, ALT, AFP, 腹部超音波 |
      | 其他檢驗項目 | CBC, DC, BUN, Creatinine, Total bilirubin, Direct bilirubin, PT, APTT, Albumin |
    And 點擊分類列標題可切換收攬/展開

  Scenario: 區塊佈局為左50%右50%
    Given 醫療人員檢視肝炎追蹤區塊
    Then 左側 50% 應顯示表格
    And 右側 50% 預留給檢驗歷程或報告內容顯示

  Scenario: 顯示肝炎追蹤項目表格
    Given 病人 "王小明" 有肝炎相關檢驗紀錄
    When 醫療人員檢視肝炎追蹤區塊
    Then 應顯示標題為「肝炎追蹤項目」的表格
    And 表格應包含以下欄位：「項目」、「值」、「參考值」、「日期」、「檢驗歷程」
    And 項目欄位應顯示項目名稱與單位，例如 "AST (U/L)"

  Scenario: CBC 與 DC 項目顯示詳細內容
    Given 病人有 CBC 或 DC 檢驗紀錄
    When 醫療人員檢視肝炎追蹤區塊
    Then 該項目的「值」欄位應顯示「詳細內容」
    And 點擊「詳細內容」後應於右側區塊顯示該項目細項（如 WBC、RBC、Hb、PLT 等）
    And 檢驗歷程中每筆紀錄的「值」欄位應顯示「點擊檢視」
    And 點擊「點擊檢視」後應於同區塊（右側）顯示該筆紀錄的詳細內容

  Scenario: 值與參考值異常時顯示警示顏色
    Given 病人檢驗值超過參考值，或定性項目為陽性/有異常值
    When 醫療人員檢視肝炎追蹤區塊
    Then 該「值」應以警示顏色顯示（如紅色或橘色）
    And 檢驗歷程列表中的值若異常，亦應以警示顏色顯示

  Scenario: 腹部超音波項目顯示檢視報告
    Given 病人有腹部超音波檢驗紀錄
    When 醫療人員檢視肝炎追蹤區塊
    Then 腹部超音波項目的「值」欄位應顯示「檢視報告」
    And 點擊「檢視報告」後應於右側區塊顯示報告內容
    And 報告內容應以可捲動的文字框顯示
    And 檢驗歷程應顯示醫令代碼（19001B、19001F、19009、19009A）及值、日期

  Scenario: 檢驗項目無資料時顯示破折號
    Given 病人 "王小明" 的 AST 項目於資料庫中無檢驗紀錄
    When 醫療人員檢視肝炎追蹤區塊
    Then AST 項目的「值」欄位應顯示 "-"
    And AST 項目的「日期」欄位應顯示 "-"

  Scenario: 點擊檢驗歷程顯示歷程列表
    Given 病人 "王小明" 的 ALT 項目有多筆檢驗紀錄
    When 醫療人員點擊 ALT 項目的「檢驗歷程」欄位
    Then 應於區塊右側 50% 顯示歷程列表
    And 歷程列表應包含「值」、「參考值」與「日期」欄位
    And 歷程應依日期排序（最新在前或最舊在前，依業務需求）
```

## 功能六：個案管理邏輯提示

### 情境：依邏輯判斷顯示收案與追蹤建議

```gherkin
Feature: 個案管理邏輯提示
  作為醫療人員
  我希望系統能依檢驗結果與管理代碼歷程自動判斷並顯示建議
  以便適時進行收案或追蹤提醒

  Background:
    Given 已成功檢索並顯示病人基本資料
    And 個案管理邏輯提示顯示於「個案管理代碼開立歷程」區塊上方

  Scenario Outline: 建議收案提示 - 管理代碼歷程為空且檢驗陽性
    Given 個案管理代碼開立歷程為空白（無任何紀錄）
    And <檢驗條件>
    When 醫療人員檢視個案管理邏輯提示區
    Then 應顯示：「個案未曾在本院收案，建議收案」

    Examples:
      | 檢驗條件 |
      | HBsAg 最近一筆為陽性 |
      | Anti-HCV 最近一筆為陽性 |
      | HCV RNA 有異常值 |
      | HBsAg 最近一筆為陽性 且 Anti-HCV 為陽性 |
      | HBsAg 最近一筆為陽性 且 HCV RNA 有異常值 |
      | Anti-HCV 為陽性 且 HCV RNA 有異常值 |

  Scenario: 建議收案 - 多條件皆符合
    Given 個案管理代碼開立歷程為空白
    And HBsAg 最近一筆為陽性
    And Anti-HCV 最近一筆為陽性
    When 醫療人員檢視個案管理邏輯提示區
    Then 應顯示：「個案未曾在本院收案，建議收案」

  Scenario: 初次追蹤個案 - 建議返診（p4201，滿3個月）
    Given 最新歷程代碼為 "p4201"
    And 該代碼開立日期距今已滿 3 個月
    And 開立日期距今未超過 6 個月
    When 醫療人員檢視個案管理邏輯提示區
    Then 應顯示：「初次追蹤個案，建議返診追蹤」

  Scenario: 初次追蹤個案 - 逾期追蹤（p4201，超過6個月）
    Given 最新歷程代碼為 "p4201"
    And 該代碼開立日期距今已超過 6 個月
    When 醫療人員檢視個案管理邏輯提示區
    Then 應顯示：「初次追蹤個案逾期追蹤, 下次可從新收案時間為 (初次開立1年之後)」
    And 提示中應顯示具體日期（初次開立日期 + 1 年）

  Scenario: 長期追蹤個案 - 建議返診（p4202，滿6個月）
    Given 最新歷程代碼為 "p4202"
    And 該代碼開立日期距今已滿 6 個月
    And 開立日期距今未超過 1 年
    When 醫療人員檢視個案管理邏輯提示區
    Then 應顯示：「長期追蹤個案, 建議返診追蹤」

  Scenario: 長期追蹤個案 - 逾期追蹤（p4202，超過1年）
    Given 最新歷程代碼為 "p4202"
    And 該代碼開立日期距今已超過 1 年
    When 醫療人員檢視個案管理邏輯提示區
    Then 應顯示：「長期追蹤個案於期追蹤, 下次可能新收案時間為 (最後一次開立1年之後)」
    And 提示中應顯示具體日期（最後一次開立日期 + 1 年）

  Scenario: 無提示 - 管理代碼歷程為空但檢驗皆陰性
    Given 個案管理代碼開立歷程為空白
    And HBsAg 最近一筆為陰性
    And Anti-HCV 最近一筆為陰性
    And HCV RNA 無異常值
    When 醫療人員檢視個案管理邏輯提示區
    Then 不應顯示「建議收案」之提示
```

## 功能七：提示優先順序與顯示規則

```gherkin
Feature: 提示顯示優先順序
  作為醫療人員
  我希望系統能正確處理多種條件同時符合時之提示顯示
  以便獲得最適切的建議

  Scenario: 提示優先順序說明
    Given 系統可能同時符合多項提示條件
    When 判斷應顯示之提示
    Then 應依以下優先順序顯示（高優先覆蓋低優先）：
      | 優先順序 | 條件 | 提示內容 |
      | 1 | 管理代碼歷程為空 且 (HBsAg陽性 OR Anti-HCV陽性 OR HCV RNA異常) | 個案未曾在本院收案，建議收案 |
      | 2 | 最新代碼 p4201 且 開立距今 > 6個月 | 初次追蹤個案逾期追蹤... |
      | 3 | 最新代碼 p4202 且 開立距今 > 1年 | 長期追蹤個案於期追蹤... |
      | 4 | 最新代碼 p4201 且 開立距今 >= 3個月 | 初次追蹤個案，建議返診追蹤 |
      | 5 | 最新代碼 p4202 且 開立距今 >= 6個月 | 長期追蹤個案, 建議返診追蹤 |
```
